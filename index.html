POWER AUTOMATE SETUP GUIDE
FLOW 1 — PARSE MEETING INVITES
When the requester submits the Excel macro, a calendar invite is sent to Civil_TQC_Calendar [at] pondco.com. This flow parses it and writes a row to the TQC_Projects SharePoint list. The dashboard reads from that list automatically.
1
Trigger: When a new event is added to calendar
Office 365 Outlook → When a new event is created (V3)
Calendar ID: Civil_TQC_Calendar [at] pondco.com
2
Parse Subject Line
Extract project number, name, review phase from:
PROJECT-2024 - Site Survey 50% Phase 1 (Redline - Revision B)
3
Parse Body Fields
Body: PM: John Smith, HPD: 11/15/2025, State: GA, Budgeted Hours: 40
Use AI Builder or regex expressions to extract all four fields.
4
Extract Reviewer from Required Attendees
Filter out Civil_TQC_Calendar [at] pondco.com. Check remaining emails against TQC_Reviewers list. If unrecognized, create a new reviewer record automatically.
5
Write to SharePoint TQC_Projects List
SharePoint → Create item
Site: your SharePoint site URL
List: TQC_Projects
Status: Queued, LoggedHours: 0, MilestoneFlag: 0
6
Send Team Notification + Requester Acknowledgment
Office 365 → Send an email (V2)
To TQC team distro + reply to organizer confirming receipt.
FLOW 2 — MILESTONE EMAILS
Triggered when the dashboard writes a MilestoneFlag value to the SharePoint list (50 or 100). Fires an email to the PM automatically.
1
Trigger: SharePoint item modified
SharePoint → When an existing item is modified
List: TQC_Projects
2
Condition: Check MilestoneFlag
If MilestoneFlag equals 50 or 100 → send email. Otherwise terminate.
3
Send Email to PM (OrganizerEmail column)
50%: Half of budgeted hours logged.
100%: Budget reached — reviewer will follow up with comments.
4
Reset MilestoneFlag to 0
SharePoint → Update item — set MilestoneFlag back to 0 to prevent re-triggering.

4
Manage the Review Queue
Go to the Review Queue tab to see all active projects. Use the action buttons on the right of each row to advance the workflow.
▶ StartMoves project from Queued to In Review. Enables hour logging.
AssignAssign a reviewer to an unassigned project.
＋ LogLog hours against the review. Only available while In Review.
✓ DoneMarks the review as Done — Pending Archive. Row turns green.
✕On a Done project: archives it. On any other: permanently deletes it.
5
Logging Hours & Milestone Emails
Click ＋ Log on any In Review project. Enter hours, date, and an optional note. Milestones trigger automatic emails via Power Automate:
50%
Halfway notification sent to the PM (OrganizerEmail). Dashboard writes MilestoneFlag=50 to SharePoint → Power Automate fires email.
100%
Budget reached — completion email sent. MilestoneFlag=100 triggers the flow.
>100%
Over-budget alert fires in dashboard. Additional hours still tracked.
💡 See the PA Setup Guide tab for configuring the Power Automate milestone email flow.
6
Overview Dashboard & Archive
The Overview tab shows live speedometer gauges for every In Review project, a portfolio-wide gauge, stat cards, and a queue preview. Archived reviews appear in the Archive tab with over/under budget analysis.
